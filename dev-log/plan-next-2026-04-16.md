# Plan: Auth Bug Fix + Remaining Hardening

**Date:** 2026-04-16
**Status:** Pending

## Priority 1: Fix Auth 401 Bug

Login succeeds (200 OK) but YouTube/Ideas endpoints return 401 immediately after. Interceptor fix applied (no longer auto-logs out), but root cause not resolved.

### Investigation Steps
1. Restart backend, login again, check terminal for `WARNING:diarynews.auth` lines
2. If "JWT invalid: ..." → likely `JWT_SECRET` not set or mismatch
3. If no warning at all → token not reaching `get_current_user` (header/proxy issue)
4. Test manually: `curl -H "Authorization: Bearer <token>" http://localhost:8000/api/youtube`

### Likely Causes
- `JWT_SECRET` not set in `.env` (backend will now crash on startup if missing — confirms/eliminates this)
- Token not attached to request (check browser DevTools → Network → YouTube request → Request Headers)
