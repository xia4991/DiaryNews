- Local embeddings (`all-MiniLM-L6-v2` via ChromaDB default) instead of API — free, no cost per document
- 800-char chunks with 200-char overlap
- `APIRouter` pattern — mountable in DiaryNews via 2 lines in `backend/api.py`

### Chinese News Tab — Full Implementation

Replaced the "News" tab with "中文新闻". Same 6 Portuguese RSS sources, all articles now get Chinese title + refined Chinese content via a second LLM call per article.

#### Task 1: DB Schema + Storage

**`backend/database.py`:**
- Added `title_zh TEXT` and `content_zh TEXT` columns to `articles` table schema
- Added `_migrate()` function with `ALTER TABLE ADD COLUMN` for existing databases (idempotent, catches `OperationalError` if columns already exist)
- Called `_migrate(conn)` at end of `init_db()`

**`backend/storage/news.py`:**
- Updated `_bulk_upsert_articles` INSERT to include `title_zh` and `content_zh` with `.get()` defaults

#### Task 2: Chinese Prompt + Parser

**`backend/prompts.py`:**
- Added `article_chinese_prompt(title, content)` — instructs LLM to translate Portuguese title to Chinese and produce a refined Chinese version of the article (not literal translation, not summary — keeps essential info, removes filler)
- Output format: `TITLE_ZH: <title>\nCONTENT_ZH: <content>`

**`backend/news.py`:**
- Added `_parse_chinese_response(text)` — extracts `title_zh` and `content_zh` from structured LLM output, handles multiline `CONTENT_ZH`

#### Task 3: Modified Enrichment Pipeline

**`backend/news.py` — `_enrich_article()`:**
- Now makes 2 LLM calls per article (was 1):
  1. Portuguese summary (`max_tokens=180`) — unchanged
  2. Chinese title + refined content (`max_tokens=600`) — new
- Returns `title_zh` and `content_zh` in the article dict
- With 1s sleep per call and 8 workers, each article takes ~4s (2 API calls × 2s each)
