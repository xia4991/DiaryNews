## Bug Fix: 401 Logout Cascade

**Problem:** After successful Google login, YouTube/Ideas endpoints returned 401. The response interceptor cleared the token on any 401, logging the user out immediately.

**Fix:**
- Response interceptor now only clears token on 401 from `/auth/me` (token validation endpoint)
- Added JWT decode logging in backend to surface root cause
- Added startup validation: `RuntimeError` if `JWT_SECRET` is missing

**Status:** Interceptor fix applied. Root cause of YouTube/Ideas 401 still under investigation — check backend logs for `JWT invalid: ...` or `JWT expired` warnings.

## Bug Fix: JWT Subject Type (resolved)

**Root cause:** PyJWT requires the `sub` claim to be a string. `create_jwt()` was passing `user_id` as an integer, causing `decode_jwt()` to raise `InvalidTokenError: Subject must be a string`.

**Fix:**
- `backend/auth.py` — `create_jwt()`: changed `"sub": user_id` → `"sub": str(user_id)`
- `backend/api.py` — `get_me()`: changed `storage.get_user_by_id(user["sub"])` → `storage.get_user_by_id(int(user["sub"]))`

**Status:** Auth fully working. Login → Google popup → JWT issued → YouTube/Ideas accessible → admin fetch buttons visible.
